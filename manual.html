01

Visao Geral

O AtlasFlow e uma aplicacao para organizar processos administrativos por usuario, etapa, prazo, prioridade e historico.

Onde esta?Status e coluna do kanban.
Com quem esta?Responsavel, origem e destino.
O que precisa?Documentos, observacoes e pendencias.
Qual prazo?Data limite e marcador de atraso.
02

Usuarios

Cada usuario representa uma pessoa, mesa ou setor. Os processos ficam vinculados ao usuario ativo.

  1. Use o seletor Alternar usuario para trocar a mesa ativa.
  2. Clique em Novo usuario para cadastrar outro setor ou operador.
  3. Ao cadastrar um processo, ele sera salvo somente para o usuario ativo.
  4. Kanban, indicadores, lista e historico exibem somente dados do usuario selecionado.
03

Primeiros Passos

Abra o sistema e confira se a tela inicial carregou com indicadores, kanban e lista de processos.

  1. Abra o arquivo index.html ou acesse o endereco publicado no GitHub Pages.
  2. Use Novo processo para cadastrar com detalhes.
  3. Use Entrada rapida quando quiser registrar agora e completar depois.
  4. Use Exportar JSON ao fim do dia para manter backup.
Importante: nesta V1, os dados ficam salvos no navegador usado. Se abrir em outro computador ou navegador, a base sera diferente.

04

Cadastro de Processo

O cadastro completo concentra identificacao, controle, prazo, saida, documentos e pendencias.

Identificacao

Numero, ano, objeto, secretaria e responsavel.

Controle

Data de chegada, origem, finalidade, prazo, status e prioridade.

Documentos

DFD, ETP, TR, Edital, Anexos e Pesquisa de mercado.

Saida

Data de saida, destino e finalidade do encaminhamento.

Exemplo de observacao objetiva:

Falta anexar pesquisa de mercado atualizada e ajustar justificativa do quantitativo no TR.

Clique em um cartao para editar o processo. Use os filtros para reduzir o kanban a uma secretaria, prioridade, documento ou termo de busca.

06

Movimentacao

Cada mudanca importante deve ser registrada pelo botao Movimentar.

  1. Localize o processo na lista.
  2. Clique em Movimentar.
  3. Informe data, novo status, destino, finalidade e observacao.
  4. Salve para atualizar kanban, lista, indicadores e historico.
Quando o status for Devolver ou Concluir, o sistema registra tambem os dados de saida.

08

Prazos e Atrasos

Um processo fica atrasado quando tem prazo vencido e ainda nao esta concluido.

  • Preencha o prazo sempre que existir SLA interno.
  • Revise atrasados no inicio do expediente.
  • Movimente o processo sempre que sair da sua mesa.
  • Conclua processos finalizados para limpar a fila operacional.

10

Rotina Recomendada

Uma rotina simples deixa o controle confiavel sem pesar no dia a dia.

  1. Abrir o AtlasFlow no inicio do expediente.
  2. Conferir indicadores de atrasados e urgentes.
  3. Registrar novas entradas.
  4. Movimentar processos que sairam da mesa.
  5. Atualizar observacoes e pendencias relevantes.
  6. Exportar backup ao fim do dia.
11

Solucao de Problemas

Algumas duvidas comuns da primeira versao local.

Os dados sumiram

Verifique se voce abriu em outro navegador ou computador. Se tiver backup, use Importar JSON.

O processo nao aparece

Clique em Limpar para remover filtros ativos e exibir todos os processos.

O processo aparece como atrasado

Confira o prazo. Se ja foi finalizado, movimente para Concluir.

Quero usar em equipe

Na V1 cada navegador tem sua propria base. Para equipe, use exportacao/importacao ou evolua para a futura versao web.
